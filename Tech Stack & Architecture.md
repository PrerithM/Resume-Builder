# **Technology Stack & Architecture**

**Project Name:** Profile Builder

## **1\. Core Framework**

* **React Native (Expo Managed Workflow):** The fastest way to build cross-platform mobile apps. Expo provides out-of-the-box libraries for all the native device features we need without touching native iOS/Android code.

## **2\. State & Form Management**

* **React Hook Form:** Crucial for a single-page app with 10+ inputs and dynamic arrays (tables). It minimizes component re-renders when the user types, ensuring the app remains lightning-fast even on older devices.  
* **useFieldArray (from React Hook Form):** Used specifically to handle the dynamic tabular data for General and Technical Education (adding/removing rows seamlessly).

## **3\. Styling & UI**

* **NativeWind (Tailwind CSS for React Native):** Allows us to implement the black/blue theme rapidly using utility classes (e.g., bg-black, text-white, border-blue-500).  
* **expo-font:** Used to load the custom Google Sans .ttf files from the local assets folder before the app renders.

## **4\. Native APIs & Device Capabilities**

* **expo-image-picker:** To access the device's photo gallery and retrieve the local URI of the user's chosen profile photo.  
* **expo-print:** The core engine for PDF generation. We will construct a template literal string of standard HTML/CSS, inject the React state variables into it, and pass it to Print.printToFileAsync().  
* **expo-sharing:** Takes the local file URI generated by expo-print and invokes the native OS share sheet (AirDrop, WhatsApp, Save to Files, Email, etc.).

## **5\. Development Steps (Roadmap)**

1. **Init & Assets:** Initialize Expo app, configure NativeWind, and load the Google Sans font locally.  
2. **Form Shell:** Setup React Hook Form and create reusable custom input components (TextInput, DatePicker, TextArea) styled with the black/blue theme.  
3. **Image Logic:** Implement the circular image upload button using expo-image-picker.  
4. **Dynamic Tables:** Implement useFieldArray for the education sections so users can map out their qualifications.  
5. **PDF Engine:** Write the HTML string template and map the form data into it.  
6. **Export Logic:** Wire up expo-print and expo-sharing to the final "Generate" button.
